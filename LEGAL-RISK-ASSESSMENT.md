## 项目原创性声明

### 技术选型

etools 是一个**独立开发**的桌面生产力启动器，采用以下现代技术栈：

| 层级 | 技术选型 | 说明 |
|------|---------|------|
| 框架 | Tauri 2.0 | 跨平台桌面应用框架 |
| 前端 | React 19 + TypeScript 5.8+ | UI 框架和编程语言 |
| 后端 | Rust 1.75+ | 系统级编程语言 |
| 搜索 | Fuse.js 7.0 | 模糊搜索算法 |
| 构建 | Vite 7 | 现代构建工具 |

**独立性说明**：
- ✅ 完全独立的技术选型
- ✅ 所有代码均为团队原创开发
- ✅ 未参考或使用任何第三方专有代码

### 开发过程

#### 第一阶段：技术调研（2025年12月）
- 研究桌面应用开发框架（Tauri vs Electron）
- 学习 Rust 和 Tauri 最佳实践
- 设计应用架构和数据模型

#### 第二阶段：核心功能开发（2025年12月-2026年1月）
- 实现 Tauri 命令系统
- 开发前端搜索界面
- 集成文件系统监控
- 实现剪贴板历史

#### 第三阶段：插件系统（2026年1月）
- 设计插件 API
- 开发插件管理器
- 创建插件市场

### 设计理念

etools 的设计遵循以下原则：
1. **简洁性**: 最小化 UI，最大化效率
2. **可扩展性**: 插件架构支持无限扩展
3. **性能优先**: Rust 后端确保毫秒级响应
4. **隐私保护**: 本地优先，数据不上云

### 贡献者协议

通过向 etools 项目贡献代码，您同意：

1. **原创性声明**: 您贡献的代码为您原创或有权贡献
2. **许可证授权**: 您的代码将采用 MIT License
3. **专利授权**: 您授予项目永久、免费的专利许可

### 代码审查标准

所有贡献的代码需通过以下审查：

1. **原创性检查**: 确保未抄袭第三方代码
2. **许可证检查**: 确保依赖库许可证兼容
3. **质量检查**: 通过所有测试和代码审查

### 贡献者流程

```bash
# Fork 项目
# 创建特性分支
git checkout -b feature/your-feature

# 提交代码
git add .
git commit -m "feat: 添加某功能"

# 推送到你的 Fork
git push origin feature/your-feature

# 创建 Pull Request
```
